Analytical Exercises

Problem Statement
Calculate the updated parameter weight for a simple linear model using one step of Gradient Descent, given a single training data point.
Analysis
To optimize the weight, we must calculate the partial derivative of the Mean Squared Error (MSE) loss function with respect to the weight parameter. This derivative (the gradient) dictates the direction of steepest ascent. We subtract the gradient, scaled by the learning rate, from the initial weight to step downhill toward the minimum loss.
Calculations
Given:
Data Point: (x = 2, y = 5)
Model: f(x) = w * x
Initial Weight: w_old = 1.0
Learning Rate: η = 0.1

1. Calculate Prediction:
f(2) = 1.0 * 2 = 2.0

2. Calculate Gradient of MSE Loss:
∇L = 2 * (prediction - true_y) * x
∇L = 2 * (2 - 5) * 2
∇L = 2 * (-3) * 2 = -12

3. Update Weight:
w_new = w_old - (η * ∇L)
w_new = 1.0 - (0.1 * -12)
w_new = 1.0 + 1.2 = 2.2
Interpretation
The initial weight under-predicted the target (2 vs 5). Consequently, the calculated gradient was negative (-12). By stepping in the opposite direction of the gradient, the algorithm correctly adjusted the weight upward to 2.2, moving the model closer to the true mathematical relationship of the data.

B
Conceptual Exercises

Definition
The Loss Landscape is a multi-dimensional topological surface where each point represents a specific combination of model parameters (weights/biases), and the height/elevation at that point represents the model's error (loss) on the training data.
Explanation
Machine learning is fundamentally a geometric search problem. Because high-dimensional landscapes contain millions of parameters, we cannot brute-force the solution. Instead, algorithms like Gradient Descent analyze the local curvature of this landscape and take iterative mathematical steps downhill, attempting to avoid "saddle points" and locate the global minimum basin where error is lowest.
Real-world Example
Imagine being dropped blindfolded onto a mountain range and tasked with finding the deepest valley. You cannot see the terrain, but you can feel the slope of the ground beneath your feet. By constantly stepping in the direction that feels steepest downward (the negative gradient), you eventually reach the bottom.

C
Computational Exercises

Objective
Implement a Gradient Descent optimization loop entirely from scratch using raw Python and NumPy arrays, bypassing high-level libraries to prove an understanding of underlying vector calculus.
Algorithm
1. Initialize weight vector randomly.
2. Compute dot product of inputs and weights (Forward Pass).
3. Compute the error vector against ground truth.
4. Calculate partial derivatives (Gradient).
5. Update weights by subtracting (learning_rate * gradient).
6. Repeat until convergence.
Python Code
import numpy as np

def gradient_descent_raw(X, y, lr=0.01, epochs=100):
    m = X.shape[0]
    weights = np.zeros(X.shape[1])
    
    for i in range(epochs):
        # Matrix multiplication for predictions
        predictions = np.dot(X, weights)
        
        # Compute gradient of MSE
        errors = predictions - y
        gradient = (2/m) * np.dot(X.T, errors)
        
        # Update weights
        weights -= lr * gradient
        
    return weights

Observation
The loss curve demonstrates a rapid, exponential decay during the first 20 iterations as the algorithm escapes the random initialization phase. It subsequently smooths into an asymptote as it approaches the optimal parameter limits, proving successful mathematical convergence.

Unit Reflection

Unit 1 fundamentally changed how I view algorithmic design. Prior to this, machine learning seemed like a black box of library calls. By breaking down the mathematics of hypothesis spaces and actually coding the calculus behind Gradient Descent, I learned that learning is strictly an optimization geometry problem.


The main challenge I faced was bridging the theoretical math into vectorized NumPy arrays. Understanding the transpose logic required to compute the gradient via matrix multiplication took significant debugging. Furthermore, experimenting with learning rates showed me firsthand how an aggressive step size causes the model to bounce erratically and diverge, rather than converge smoothly.


These foundational principles have direct applications to everything I build moving forward. Whether I am tuning a simple linear regressor or eventually building deep neural networks, the rules of the Loss Landscape remain the same: you must mathematically audit your cost function, or your algorithm will fail.
